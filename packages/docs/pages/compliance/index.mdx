# Compliance & Security

Recuring is built for Indian businesses and complies with applicable Indian data protection and privacy laws.

- **[Privacy Policy](/compliance/privacy)** — How we collect, use, and protect your data.
- **[Terms of Service](/compliance/terms)** — The agreement governing your use of Recuring.
- **[DPDP Compliance](/compliance/dpdp)** — Your rights under the Digital Personal Data Protection Act 2023.
- **[Grievance Officer](/compliance/grievance)** — How to raise a complaint or data request.

## Data Residency

All customer data is stored on servers located in **India** (AWS ap-south-1, Mumbai region). No personal data is transferred outside India except where required by law or with explicit consent.

## Security Overview

| Control | Details |
|---|---|
| Encryption in transit | TLS 1.2+ on all connections |
| Encryption at rest | AES-256 (database and file storage) |
| Authentication | Email + password with bcrypt hashing; optional 2FA |
| Access control | Role-based (Admin, Staff, Technician, Customer) |
| Database backups | Point-in-time recovery, weekly restore verification |
| Vulnerability management | Dependency scanning on every CI build |
| Penetration testing | Annual third-party assessment |

## Incident Response

In the event of a security incident affecting personal data, we follow a defined incident response process:
- Containment within 2 hours of detection
- Notification to affected users within 72 hours
- Root cause analysis and remediation report
